# Guía de contribución

Gracias por colaborar con este proyecto.

## Antes de enviar cambios

- Revisa el contexto del repositorio y su propósito pedagógico.
- Mantén el código en inglés.
- Mantén los comentarios y la documentación en español neutro.
- Evita introducir dependencias innecesarias.

## Flujo sugerido

1. Crea una rama para tu cambio.
2. Implementa el ajuste con pruebas cuando aplique.
3. Ejecuta `mvn test`.
4. Actualiza `README.md` si cambió la forma de uso o la estructura del proyecto.
5. Abre un Pull Request con una descripción breve y clara.

## Criterios de calidad

- El proyecto debe compilar con Maven.
- Los nombres del código deben ser consistentes y descriptivos en inglés.
- La documentación debe seguir un tono claro y neutral en español.
- Las demos deben conservar su intención educativa.
